# SPEC — Portfolio Hub

> 손영선(신입 풀스택/백엔드 개발자, GitHub `jakesoneyo`)의 **개인 포트폴리오 허브 랜딩 페이지**.
> 이 사이트 자체가 포트폴리오이며, 포폴 공장에서 나오는 프로젝트들을 모아 보여주는 관문(hub) 역할을 한다.

- **난이도 티어:** S (하루 완성)
- **유형:** 정적 프론트엔드 랜딩 (일반 유형 태그 CRUD/실시간/인증 등에 해당 없음 → 다양성 규칙 미적용)
- **스택:** Vite + React + TypeScript + Tailwind v4 (프론트 전용, 백엔드/DB/API 없음)
- **배포:** Vercel

---

## 1. 문제 정의

채용 담당자·면접관이 손영선의 역량과 프로젝트를 **한 페이지에서 빠르게 파악**할 수 있어야 한다.
동시에 이 워크스페이스(`포폴/`)는 프로젝트를 계속 찍어내는 공장이므로, 허브는 **새 프로젝트가 완성될 때마다 코드 수정 없이 데이터 파일에 항목만 추가하면 카드가 늘어나야** 한다. 이 "데이터 주도 확장성"이 프로젝트의 핵심 설계 요구사항이다.

## 2. 핵심 기능 범위 (In Scope)

| # | 섹션 | 내용 |
|---|------|------|
| 1 | Hero / Intro | 이름·한 줄 소개, 자기소개 문구, 연락 CTA(이메일/GitHub) |
| 2 | 핵심 역량 (Skills) | Frontend / Backend / Data·Deploy 3개 카테고리별 기술 태그 |
| 3 | 프로젝트 목록 (Projects) | `src/data/projects.ts` 데이터 파일을 map으로 렌더링한 카드 그리드. 지금은 비어있거나 "Coming soon" 상태 |
| 4 | 연락처 (Contact) | 이메일 `jakesoneyo@gmail.com`, GitHub `github.com/jakesoneyo` 링크만 |

**공통 요구사항 (S 티어 정적 사이트 기본기):**
- 반응형 (모바일 / 태블릿 / 데스크톱)
- 접근성 기본 (시맨틱 태그, alt, 키보드 포커스, 충분한 색 대비, 스킵 링크)
- 부드러운 앵커 스크롤 네비게이션 (한 페이지 단일 스크롤)
- 프로덕션 빌드 + Vercel 배포 + README

## 3. 명시적 비범위 (Out of Scope)

- 백엔드 / DB / API / 인증 / Swagger / Testcontainers / DB 마이그레이션 → **전부 N/A** (백엔드 없음)
- 전화번호 노출 (사람이 명시적으로 제외 결정)
- 실무 경력 프로젝트(유현건설 웹/앱, 스트롱쌀롱) → 이번 스코프 제외. 아래 "향후 확장 후보"로만 기록
- CMS / 어드민 / 다국어(i18n) / 다크모드 토글 / 블로그 / 애널리틱스 → 과설계, 하지 않음

## 4. 유저 스토리

- **면접관으로서** 사이트 첫 화면에서 이 사람이 무엇을 하는 개발자인지 5초 안에 파악하고 싶다.
- **면접관으로서** 기술 스택을 카테고리별로 한눈에 보고 우리 팀 니즈와 매칭하고 싶다.
- **면접관으로서** 각 프로젝트의 라이브 데모/GitHub로 바로 이동하고 싶다.
- **채용 담당자로서** 모바일에서도 깨지지 않게 보고, 이메일/GitHub로 즉시 연락하고 싶다.
- **손영선(운영자)으로서** 새 프로젝트를 완성하면 `projects.ts`에 객체 하나만 추가해 카드를 늘리고 싶다 (컴포넌트 수정 없이).

## 5. 성공 기준 (Definition of Done)

- [ ] 4개 섹션(Intro/Skills/Projects/Contact)이 한 페이지에 렌더링된다.
- [ ] 프로젝트 카드가 **오직 `src/data/projects.ts`** 로부터 생성된다 (하드코딩된 카드 없음).
- [ ] 데이터가 비어도 빈 상태("포트폴리오 프로젝트 준비 중" 등) UI가 정상 표시된다.
- [ ] 데이터 배열에 항목 1개를 추가하면 컴포넌트 코드 변경 없이 카드가 1개 늘어난다.
- [ ] 모바일(≈375px)·데스크톱(≥1024px)에서 레이아웃이 깨지지 않는다.
- [ ] 키보드 Tab만으로 모든 링크에 도달·활성화 가능, 포커스 링이 보인다.
- [ ] `npm run build` 성공, Vercel 프로덕션 배포 URL이 열린다.
- [ ] README에 실행법·배포법·`projects.ts` 항목 추가 방법이 문서화된다.

## 6. 향후 확장 후보 (지금 구현 안 함)

- **경력(Experience) 섹션:** 유현건설 웹페이지/어플, 스트롱쌀롱 등 실무 프로젝트. 현재 보류 → 추후 `src/data/experience.ts` 같은 별도 데이터 파일 + 섹션으로 추가 가능.
- 공장에서 나오는 각 포폴 프로젝트가 완성될 때마다 `projects.ts`에 항목 추가 (설계상 이미 지원, 확장이라기보다 정상 운영 흐름).
- 프로젝트 상세 페이지(라우팅) — 지금은 외부 링크로 충분하므로 미도입.
- 다크모드, i18n — 필요성이 확인되면 그때 도입.
